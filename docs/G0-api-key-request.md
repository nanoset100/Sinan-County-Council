# G0 — S1 Open API 인증키 발급 및 커버리지 실증 (PRD v0.4 §12-G0)

> **이 게이트는 PRD 확정 조건이다.** 통과 전까지 §6 파이프라인은 잠정("조건부 확정")이며,
> 파이프라인은 `COLLECTOR=manual` 폴백으로만 동작한다.

## 현재 상태 (2026-07-02 웹 실증)

- ✅ 국회도서관 지방의정포털(clik.nanet.go.kr) Open API 존재 확인. **243개 지방의회 연계(군의회 82곳 포함)** → 신안군 포함 개연성 높음.
- ✅ 제약 확인: **호출당 100건, 인증키당 일 1,000회**. 인증키 양도 불가.
- ❌ **블로커: 인증키 발급이 사람의 행정 절차(메일 신청, 영업일 7일 내 회신)라 자동 완료 불가.**
- ⛔ 신안군 `rasmblyId` 실호출 실증은 인증키 없이는 불가.

## 🔴 G0 판정 결과 (2026-07-02): 전면 수기 주도형 확정

포털(웹) 및 Open API 실사 결과, **MVP 대상인 신안군 제10대(2026-07~) 데이터가 S1 에 없음**을 확인:

| 확인 항목 | 결과 |
|---|---|
| Open API rasmblyId 목록에 신안군 | ❌ 부재 (전남은 고흥·광양·나주·목포·순천·여수·영광 7곳만) |
| 웹 상세검색 — 신안군 **의안** (전 기간·전 대수) | ❌ 0건 |
| 웹 상세검색 — 신안군 **회의록** | ⚠️ 168건이나 **전부 제6대(2013) 과거분**, 제10대 미반영 |
| 포털 공지 | "제9회(2026) 지선 데이터는 각 지방의회 홈페이지 참고해 추후 업데이트 예정" |

**결정:** §6 을 **수기 주도형(폴백)** 으로 확정. 원문 정본 = 신안군의회 홈페이지(S2). `COLLECTOR=manual` 기본.
S1Collector 는 (a)과거 소급 아카이브 (b)향후 개방 시 전환을 위한 어댑터로 유지(코드는 실제 명세로 확정됨).

**후속 조치:**
1. 국회도서관/신안군의회에 **신안군 제10대 Open API 개방·데이터 반영 요청**(아래 템플릿 활용, `오류신고`/`lawstat@nanet.go.kr`).
2. 인증키가 승인되면 `npm run g0` 로 회의록 API(minutes.do) 의 신안 rasmblyId 반환 여부를 확인 → 과거분 소급에 활용 가능한지 판단.
3. PRD 를 v0.5 "확정 — 수기 주도형"으로 갱신.

> ※ 인증키 자체는 계속 발급받아 두는 것이 좋다(향후 개방·소급 아카이브 대비). 아래 신청 절차는 유효하다.

## 1단계 — 인증키 신청 (사람이 수행)

**메일 신청:** `lawstat@nanet.go.kr` 로 아래 5개 필수 항목을 보낸다.

```
받는사람: lawstat@nanet.go.kr
제목: 지방의정포털 Open API 인증키 신청

1. 신청기관: (예) 신안군 의정활동 공개 시민 정보 서비스 운영팀
2. 담당자: (이름)
3. 담당자 기관 전화번호: (연락처)
4. 신청 목적: 신안군의회 의정활동 기록(회의록·의안·의원정보)을 공개된 공식
   기록에 근거해 비영리·중립적으로 재공개하는 시민 투명성 포털 구축.
5. 신청 서비스 종류: 회의록, 의안정보, 의원정보 (5분자유발언 포함 시 명시)
```

- 문의: 국회도서관 데이터융합분석과 02-6788-4475 / 리소스센터 02-6788-4150
- 회신: 업무일 기준 7일 이내.
- 대안: 포털 로그인 후 리소스센터(리소스센터 `/potal/guide/resourceCenter.do`)에서 인증키 신청 및 개별 API 명세 확인.

## 2단계 — 신안군 rasmblyId 확인

- 리소스센터의 의회코드/기관 목록에서 **"신안군의회"** 항목의 식별자를 찾는다.
- 값을 모르면 3단계 스모크 테스트를 후보값으로 반복 실행해 반환 데이터로 확정한다.

## 3단계 — 실호출 실증 (`npm run g0`)

```bash
# .env 에 S1_API_KEY, S1_RASMBLY_ID 를 넣은 뒤
npm run g0
# 또는 후보 rasmblyId 를 직접:
npm run g0 -- --rasmbly=<후보값>
```

스크립트가 `bill.do` / `minutes.do` / `assemblyinfo.do` 를 실호출하고 HTTP 상태·응답 구조·건수·필드를 리포트한다.

## 4단계 — 판정 및 반영

- **신안군 의안·회의록이 반환됨 → G0 통과:**
  1. `.env` 의 `COLLECTOR=s1` 로 전환.
  2. `pipeline/collectors/S1Collector.ts` 의 `extractRows` / `pickSourceUrl` / `pickSourceText` 와 `pipeline/normalize.ts` 의 S1 필드 매핑을 **실제 응답 필드명으로 교정**.
  3. PRD 를 **v0.5 "확정"** 으로 갱신하고 §6 을 "S1 주도형"으로 확정.
- **404/빈 응답 → BASE_URL·파라미터 교정(리소스센터 실제 명세) 후 재실행.**
- **신안군 데이터가 끝내 없음 → §6 폴백(수기 주도형 + §5 범위 축소)으로 확정** + 국회도서관에 연계 문의 병행.
